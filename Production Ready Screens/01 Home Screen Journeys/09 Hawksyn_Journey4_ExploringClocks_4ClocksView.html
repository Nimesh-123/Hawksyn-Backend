STATE 1 — Entry · Clocks animate in, scores count up 0 → value
S01·1
Default — all 4 clocks live, entry animation playing
Live · Entry
Stagger entry
Each clock card fades in + scales up from 92% one by one. Delays: C1=50ms, C2=200ms, C3=350ms, C4=500ms. Total entry sequence = ~850ms. Feels alive, not instant.
Score count-up
Score number inside each arc counts from 0 to final value over 800ms using an ease-out curve. Arc stroke animates simultaneously — draws from left to right matching the score. User watches the clocks "settle."
Clock 1 pulse
Clock 1 (AI Exposure, HIGH RISK) has a continuous slow red border pulse — 2s loop, ease in-out. It breathes. This is the ONLY clock that pulses. Urgency signal. Other clocks are static.

STATE 4 — Long press Clock 1 (500ms hold) · Market signal peek tooltip
S01·4
Long press — market signal tooltip over Clock 1
Long press
Signal peek
Hold 500ms on any clock → dark tooltip rises from the card showing the exact market event that drove the score. No navigation. Just a peek. Release to dismiss. Gives curious users the "why" without cluttering the main view.
Tooltip content
Shows: trigger event + date detected. For C1: "3 new AI tools launched this month that handle backend API generation tasks." For C2: "Hiring in Node.js dropped 18% vs last month on Naukri + LinkedIn." Etc.
Haptic trigger
At 500ms hold → light haptic tap fires simultaneously with tooltip appearing. Confirms the long-press registered. Standard iOS/Android haptic feedback pattern. Single tap, not a buzz.
